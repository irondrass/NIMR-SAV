Cloud Supabase

Sauvegarde, restauration et réglages atelier sécurisés sans perdre les données locales

Utilisez uniquement la publishable key / clé publique Supabase côté navigateur. Ne jamais utiliser service_role ici : la sécurité dépend de l'authentification, des règles RLS par atelier et de la vérification que RLS est activé avant un usage réel.

Une copie de sécurité locale est conservée dans l’application avant restauration cloud.

Avant chaque sauvegarde cloud, l'application télécharge automatiquement une copie JSON locale de sécurité. Avant restauration, une copie locale est conservée silencieusement dans l'application.

Supabase non configuré : synchronisation cloud inactive.

Exécutez en priorité supabase_nimr_sav_v23_2_8_full_audit.sql dans SQL Editor. supabase-schema.sql reste le schéma de compatibilité historique.
